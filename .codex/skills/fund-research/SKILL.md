# 基金研究

## 适用范围

使用本技能处理以下任务：

- 比较公募基金、ETF、ETF联接、LOF、FOF、QDII、债券基金、指数基金的费用、申购额度、暂停状态、跟踪标的、业绩差异和适配场景。
- 查询或整理指数数据，例如纳斯达克100、标普500、中证红利低波动、A股红利、港股红利、央企红利等。
- 更新本地基金笔记、定投计划、组合配比、指数年度涨跌表和基金筛选表。
- 将多个基金或指数按同一口径归一化，形成可审计的对比结论。

不要把输出写成确定性买卖指令。用户要求投资建议时，给出基于假设的研究结论、风险约束和可执行检查项，而不是承诺收益或替用户决策。

纳斯达克100专题任务可读取 `references/nasdaq100.md`；如果独立的 `nasdaq100-research` 技能同时可用，优先让该技能承担纳指100专门口径。

## 资料优先级

当前金融数据必须核验，不要把本地旧笔记直接当作最新事实。

资料优先级如下：

1. 官方来源：基金公司产品页、公告、招募说明书、基金合同、定期报告、ETF申赎清单、交易所、指数公司、监管披露。
2. 理杏仁：用于指数历史涨跌、全收益口径、估值/基本面、基金和指数交叉核对。
3. 主流行情或销售平台：只作发现线索或辅助核验，关键字段仍回到官方或理杏仁确认。

如果来源冲突，优先采用官方披露，并在输出中说明冲突点、采用口径和数据日期。若无法联网或无法打开关键来源，明确标注“未能核验最新状态”，不要假装数据是实时的。

## 本地知识库

开始工作前先读取相关本地文件：

- `基金/投资圣经.md`：用户的长期观察和经验规则。
- `基金/配比.md`：用户当前组合目标、定投安排和已投品种。
- `基金/纳指100/`：纳斯达克100、NDX、QQQ/TQQQ、相关 QDII/ETF/联接基金研究。
- `基金/标普500/`：标普500指数与相关基金研究。
- `基金/红利指数/`：红利、低波、央企红利、港股红利等主题研究。

本地文件名中带日期的内容视为当日快照。引用时要说明快照日期；需要“最新”结论时必须重新核验。更新本地笔记时，优先创建或更新带日期的主题文件，并保留“版本说明”“口径说明”“来源说明”。

## 工作流

1. 确定问题范围：标的、市场、基金类型、时间区间、币种、是否只看可申购/可交易产品。
2. 读取相关本地笔记，提取用户偏好、既有结论和上次口径。
3. 核验当前数据：申购/暂停/限额、费率、跟踪标的、定期报告收益差、ETF场内交易与一级申赎状态。
4. 统一口径：区分基金份额类别、场内ETF与场外联接、价格指数与全收益指数、人民币与美元份额、自然年与年初至今。
5. 计算并复核：所有派生值写明公式；百分比和百分点分开表达；估算值必须标“估”。
6. 输出结论：先给最有用的结论，再给表格、口径、风险点和来源。
7. 如需写入文件，保持原有知识库风格，用 Markdown 表格，避免覆盖无关内容。

## 理杏仁取数口径

理财类数据优先参考各自官网数据或理杏仁。

理杏仁年度涨跌数据查询：

1. 搜索对应指数或基金。
2. 进入“月度涨跌”。
3. 指数研究优先选择“全收益率”口径；若只能取得价格指数或基金净值口径，必须注明。
4. 查看月度涨跌表最后一列的年度涨跌数据。
5. 记录页面显示的更新日期、数据截止日期和是否存在缺失行。

不要混用以下口径：价格指数 vs 全收益指数、指数本体 vs 跟踪基金、发布以来 vs 有数据以来、自然年 vs YTD、港币/美元/人民币计价。

## 基金对比字段

基金筛选表通常包括：

- 基金名称、代码、份额类别、基金类型。
- 跟踪标的或业绩比较基准。
- 当前申购/交易/暂停状态、单日额度、渠道差异。
- 管理费、托管费、销售服务费、管托合计。
- 成立日期、规模、流动性、ETF溢价率/折价率、成交额。
- 基金净值增长率、业绩比较基准收益率、长期累计收益差。
- 跟踪误差、超额收益、基金经理或产品变更等特殊事项。

常用计算：

- 管托合计 = 管理费 + 托管费。
- 长期累计收益差 = 基金份额净值增长率 - 同期业绩比较基准收益率。
- 除去管托后的长期累计收益差只能作粗估；写清假设，不把估算当作公告披露。
- QDII 产品还要关注外汇额度、境外市场休市、申购暂停、赎回到账周期和汇率口径。

## 输出格式

回答或笔记建议按这个顺序组织：

1. 结论：用几句话说明最重要发现，避免空泛建议。
2. 口径说明：列明数据日期、来源优先级、筛选条件、排除项。
3. 对比表：字段清晰，单位一致，缺失值用 `--`，估算值加“估”。
4. 风险和注意事项：申购限制、暂停风险、溢价风险、汇率风险、跟踪误差、规模和流动性。
5. 来源说明：列出官方/理杏仁来源及日期；联网检索时附链接。

更新 Markdown 文件时，表格后补充简短“来源说明”和“未纳入/口径差异”，这样未来能复核为什么当时得出该结论。

## 运行环境

运行 Python、测试、脚本或数据处理命令时必须使用 `jijin` conda 环境；优先使用 `conda run -n jijin ...`，禁止直接使用系统 `python`/`python3`/`pip`。

## 核验清单

提交前检查：

- 是否读取了相关本地笔记，并把旧快照与最新数据区分开。
- 是否有数据日期、截止日期或公告日期。
- 是否优先使用官方来源或理杏仁。
- 是否把 A/C/E/I/Y、人民币/美元、ETF/联接、场内/场外分清楚。
- 是否避免把不同指数或不同收益口径直接横向比较。
- 是否注明估算公式、缺失数据和不可核验之处。
- 是否把结论写成研究判断，而不是保证收益或强制买卖建议。
